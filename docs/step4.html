Step 4 - Generated Content



Generated content instead of static HTML



It is an immanent property of this type of application that the page holds more content than the window can display at once.
For latency reasons, initially invisible parts should not be rendered immediately.

To simplify things, content shall be completely generated (not loaded), and the logic when to show it shall be reduced to All content at once after closing the overlay.

The amount of sections shall be limited by Array pages
which also defines the respective sections' content (it will hold the URLs to load later).

Sections are created using 
document.createElement("div")
and then appended using the command 
appendChild()

Previously defined CSS classes are used so that only the properties that differ,
backgroundColor and innerHTML,
must be mentioned separately.
