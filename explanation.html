Some changes, enabling the application to react on problems potentially occuring in real life.

The page is no longer dependent of successfully loading the explanation:
fallback = setTimeout(load_pages, 3001, 0);
invokes content generation even if loading the explanation fails; if it succeeds during 3 seconds,
clearTimeout(fallback);
will delete the timeout.

AJAX will be handled more generic:
function load_content(url, element, callback_function)
will treat internet address [url], explanation content zone [element] and success action [callback_function] as parameters to make it reusable for potential other AJAX calls.
Loading will only be seen as successful if meta - tags can be found (displaying 404 messages usually is unwanted).

Content generation is only a simulation of content loading, and content loading can fail. Therefore, every section to fill is observed by
section_content.observer = new MutationObserver(dom_changed);
As long as no change is detected, the next section's generation will not be invoked.
This can be checked easily by changing the pages - Array to
var pages = [ "Part 1", "", "Part 3", "Part 4", "Part 5" ];
for testing purpose.

Finally, one of the first three sections will be expanded with dummy content to demonstrate the following sections will move.
